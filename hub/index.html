This error occurs when macOS blocks osascript or when Freedom Guard lacks permissions to manage proxy and network settings.

1. Grant Automation Permissions

  • Open System Settings ‚Üí Privacy & Security ‚Üí Automation
  • Find Freedom Guard in the list
  • Enable permissions for System Events or osascript

2. Accessibility & Full Disk Access

  • Go to System Settings ‚Üí Privacy & Security
  • In Accessibility, click + and add Freedom Guard
  • In Full Disk Access, add Freedom Guard

3. Reset Automation Permissions

Run this command in Terminal:

tccutil reset AppleEvents

Restart your Mac to request permissions again.

4. Manual Proxy / TUN Mode Setup

Proxy Mode

networksetup -setwebproxy "Wi-Fi" 127.0.0.1 8086
networksetup -setsecurewebproxy "Wi-Fi" 127.0.0.1 8086

To turn off:

networksetup -setwebproxystate "Wi-Fi" off
networksetup -setsecurewebproxystate "Wi-Fi" off

TUN Mode

In System Settings ‚Üí Privacy & Security, allow VPN / System Extension access for Freedom Guard.

5. Restart & Test

  • Completely close Freedom Guard
  • Reopen the app
  • Select System Proxy or TUN Mode
  • Test your connection

The error occurs because creating and configuring a TUN interface on Linux requires root privileges. If logs show "operation not permitted" and the app wasn‚Äôt run as root, TUN mode cannot initialize.

1. Run with Root Privileges

Try running Freedom Guard with administrator (root) privileges:

sudo ./freedom-guard-linux-x86_64.AppImage

Or, if installed:

sudo freedom-guard

This allows the TUN interface to be created properly.

2. Running in TUN Mode

To enable TUN mode, provide root access to Freedom Guard. Execute these commands one by one:

First Command

xhost +SI:localuser:root

Allows the root user to access your display for the graphical interface.

Second Command

sudo -E freedom-guard --no-sandbox

Runs Freedom Guard with root privileges. The -E preserves environment variables, and --no-sandbox disables the sandbox for full system access in TUN mode.

Third Command

xhost -SI:localuser:root

Revokes root access to your display after use for security.

3. Verify TUN Mode

  • Close Freedom Guard completely
  • Reopen using the above commands
  • Select TUN Mode
  • Test your connection

Important Notes for Windows Portable Version

  • Always run as Administrator
  • The Portable version doesn't require installation
  • Works on all Windows versions

1. Bypassing "Windows Protected Your PC" Warning

If Windows shows a "Windows protected your PC" message when running Freedom Guard:

  • Click More info
  • Select Run anyway

2. Run as Administrator

Right-click the Freedom Guard executable and select Run as administrator to ensure proper functionality.

3. Test Connection

  • Close Freedom Guard completely
  • Reopen as Administrator
  • Select your preferred mode (System Proxy or TUN Mode)
  • Test your connection
